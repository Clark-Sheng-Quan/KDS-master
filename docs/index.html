Kitchen Display System

KDS is a kitchen display application developed by vend88. It works alongside VendPOS — when a cashier places an order at the counter, it appears on the KDS screen in the kitchen immediately, without any paper tickets or verbal relay. KDS can also pair with a Calling Screen to notify customers when their order is ready.

What is KDS?

The kitchen screen that connects to VendPOS

KDS runs on an Android tablet mounted in the kitchen. It connects to VendPOS over the local network and receives orders in real time — showing each ticket on screen the moment it is placed, keeping every station informed without paper or phone calls. It can also link to a Calling Screen device, automatically pushing active orders to the customer-facing display and letting staff call individual orders to ready with a single tap.

From idle to in-progress — instantly

The moment a customer places an order, it appears on screen. No refresh needed. Orders flow in from POS terminals and online platforms, merged into one live view.

System is online and waiting

When no orders are active, the screen shows a clean idle state. Staff can confirm at a glance that the system is connected and ready. The header always shows the live order count and current time.

New Orders (0) — live clock in the top right, Recall Order button always accessible
2

First order arrives

A new order card appears the instant it's placed — showing the order type (Dine-In / Take-Away), table or order number, and the items to prepare. The header count updates immediately.

Real-time Updates

Order changes reach the kitchen immediately

If a customer modifies their order after placing it — adding an item, swapping a customization — the kitchen card updates instantly with a clear visual alert. No verbal relay, no re-printing.

The "Updated" badge
When an order is modified, an orange ↻ Updated badge appears at the top of the card. New items are added directly below the existing ones. Staff can spot the change at a glance without re-reading the entire order.
Mocha with customizations added to Dine-In #12 — the Updated badge makes the change unmissable
💡

Updates are pushed in real time via the same TCP and API connection that delivers new orders. There is no polling delay — the badge appears the moment the POS sends the change.

Completing Orders

Mark progress item by item, or all at once

KDS supports both item-level and order-level completion. Stations can tick off individual items as they finish, or mark the whole order done in one tap when everything is ready.

1

Item-level completion

Tap any item to mark it done — it highlights green with a ✓ checkmark and stays visible. This lets different stations tick off their own items independently: the coffee bar marks the Latte done while the kitchen is still working on the main course.

Latte done ✓ — Mocha's customizations being ticked off one by one as each station finishes
2

More orders arrive — each managed independently

As new orders come in, they stack into the grid. Each card is fully independent: completing one order has no effect on others. The screen stays organized no matter how many tickets are open.

Two orders in progress
Three orders — each with its own All Done button
Cancellations

Know immediately what not to make

When a customer cancels items — through the POS or online — those items are marked on the kitchen card in real time. Staff can stop preparation without any verbal communication from front-of-house.

Cancelled items stay visible
Cancelled items appear with a strikethrough and a Cancelled label rather than disappearing. This matters: if a station has already started on an item, they need to see it was cancelled — not just notice it's gone.
Tuna Avocado Roll and Sushi Makizushi cancelled — shown clearly so no wasted prep work
⚡

Why not just remove them? In a busy kitchen, items disappearing silently from a card causes confusion — staff won't know whether the card was updated or if they missed something. Keeping cancelled items visible with a clear label means every station always knows the full, current state of every order.

Recall Orders

Undo a completion in seconds

Mistakes happen — a staff member marks an order done too early, or a customer changes their mind after it's been cleared. KDS makes it easy to bring any completed order back to the active screen.

1

Online takeaway order arrives

Alongside dine-in tables, KDS receives orders from online platforms. Takeaway orders appear as their own card — same layout, same All Done button — so the kitchen handles them exactly the same way regardless of source.

Four concurrent orders — the rightmost is an online takeaway, treated identically to dine-in tickets
2

Instant recall after completion

When an order is marked All Done, a confirmation card appears in the corner showing the order number and a one-tap Recall Order button. This gives staff an immediate undo window without having to navigate anywhere.

Order completed — the recall shortcut is right there if it was a mistake
3

Recall Order panel — full history

The Recall Order button in the header opens a panel listing all recently completed orders. Each entry shows the order number, table, item count, and a summary of items. Any order can be recalled back to the active display with one tap.

Completed orders listed most-recent first — recall any of them back to the active screen instantly
💡

Recalled orders reappear on the main display exactly as they were — including all items, customizations, and order notes. No data is lost when an order is completed.

Settings · Device Info

Know exactly where your KDS is on the network

The Device Info panel shows the network identity of the KDS device at a glance — the information POS terminals and other devices need to connect to it. No network scanning or manual lookup required.

Local IP Address
Displays the device's current IP address on the local network. This is the address POS terminals are configured to send orders to. Staff or administrators can read it directly from the screen without accessing router settings.
TCP Port
KDS listens for POS order traffic on port 8080. The port is shown here so it's easy to reference when configuring a new POS terminal or troubleshooting a dropped connection.
Device Name
A unique identifier for this KDS unit on the network (e.g. KDS:edd2). Useful when multiple KDS devices are deployed across a larger kitchen or multi-station setup.
IP address, TCP port, and device name — everything needed to connect a POS terminal, visible at a glance
Settings · POS Connection

Track every POS terminal from one screen

KDS remembers every POS terminal that has ever connected over TCP. The POS Connection panel shows their current status in real time, so kitchen staff and managers can see at a glance which terminals are active and which have dropped off the network.

Live connection status per terminal
Each POS system is listed by its IP address with a live status indicator: Connected means the terminal is actively sending orders, Disconnected means it has gone offline or was powered down. The list updates automatically — no manual refresh needed.
Three POS terminals tracked simultaneously — 192.168.0.140 is active, the other two are offline
Reset Connection
Each terminal has a Reset Connection button that clears its session and forces a fresh TCP handshake. This is useful when a POS terminal reconnects from a different IP after a restart, or when a connection has stalled without fully dropping.
💡

KDS supports multiple simultaneous POS connections. Orders from all connected terminals are merged into a single live view — staff see everything in one place regardless of which terminal the order came from.

Settings · Item Click Mode

Two ways to mark items done — choose what fits your workflow

Different kitchens work differently. Some prefer tapping once to mark an item in progress; others prefer items to disappear completely when double-tapped. KDS supports both modes, switchable in Settings with no restart needed.

1

Switch to "Double Click to Remove" mode

In Settings, switching the Item Click Mode to Double Click to Remove changes how item completion works across the entire display. The setting takes effect immediately.

Double Click to Remove selected — items will disappear from the card when double-tapped rather than turning green
2

Orders adapt: "Done" button + item-level Recall

In this mode the completion button changes from "All Done" to Done, and the header gains a Recall Item button alongside the existing Recall Order button — because removing individual items now requires a separate undo path.

Done button, Recall Order, and Recall Item — the interface adapts to the mode automatically
3

Item removed — instant confirmation with Undo

Double-tapping an item removes it from the card and shows an ITEM COMPLETED toast in the corner naming the item. An Undo button is available for a few seconds in case it was a mistake.

"Tonkotsu Ramen 豚骨ラーメン" removed — Undo available immediately before the toast disappears
4

Recall Item — bring back any removed item

Tapping Recall Item in the header opens a panel listing all recently removed items. Each entry shows the order it came from, the item name, and its preparation time. Any item can be restored to its original order card.

Item-level recall history — each item shown with its order number, name, and how long it took to prepare
5

Item restored to the active order

After recalling, the item reappears on its original order card exactly as before. The order continues as if the item was never removed — no need to re-enter anything.

Settings · Card Size & Font

Fit more orders on screen, or make them easier to read

Cards Width and Cards Height control how much space each order card takes up. Font size is a separate control that can be combined freely — shrink cards to show more orders, or bump the font for stations that need to read from a distance.

1

Default — Medium card size

Out of the box, both Cards Width and Cards Height are set to Medium. This gives each order card a balanced amount of space — clear item names, readable quantities, and enough room for the All Done button — while still fitting several orders on screen at once.

Cards Width: Medium · Cards Height: Medium — the default starting point
2

Large cards — easier to read at a distance

Increasing card size makes each ticket larger and more legible — useful on screens mounted further from the prep station, or where item names are long and need more room to display cleanly.

Large card size — each order takes up more space, text is easier to read from across the station
3

Small cards — more orders visible at once

Reducing card size lets the screen fit more concurrent orders without scrolling. Useful during peak service when many tickets are open and the team needs a full overview at a glance.

Small card size — more orders visible on screen simultaneously
4

Font size is adjustable independently

Title Font Size and Item Font Size can be set separately from card dimensions. Here, both are set to Small alongside Small card Width and Height — combining compact cards with a tighter font to maximize the number of orders on screen.

Cards Width: Small · Cards Height: Small · Title Font: Small · Item Font: Small
Small font + small cards — maximum order density, minimal screen space per ticket
💡

All size and font changes take effect immediately — no restart needed. Card size and font size are independent controls, so any combination is valid.

Settings · Print

Print tickets your way — manually or automatically

KDS integrates with a thermal printer to produce kitchen tickets. Three settings control how and when printing happens: whether the print button is visible on each card, whether new orders print automatically on arrival, and how many tickets are cut per job.

Three print settings, one group
All print configuration is grouped together in Display Settings. Show Print Button toggles a manual print button on every order card. Auto-Print New Orders triggers a print job the moment an order arrives — no staff action needed. Print Mode controls whether the printer cuts once per item or once per complete order.
Show Print Button · Auto-Print New Orders · Print Mode — all three in one section
Auto-Print New Orders
When enabled, a ticket prints automatically every time a new order arrives — including orders from both POS terminals and online platforms. Staff receive a physical ticket without touching the screen, useful in kitchens where the display is shared but each station needs its own paper copy.
Auto-Print On — every incoming order triggers an immediate print job
Print Mode: Cut per Item or Cut per Order
Cut per Item prints one ticket per menu item — each item gets its own slip, useful when different stations handle different dishes and each needs its own paper. Cut per Order prints all items on one continuous ticket per order, better suited for kitchens where one station handles the full order.
Cut per Item selected — each dish gets its own ticket rather than grouping the entire order
Show Print Button — manual trigger on every card
Enabling Show Print Button adds a Print button to the header of every order card. Staff can reprint any order at any time — useful if a ticket is lost or a station didn't receive the auto-print. The button appears and disappears without any restart.
Every active order card has a Print button — reprint any ticket on demand, at any point during service

43m, 40m, 39m, 36m in red — all overdue. 05m in green — just arrived. No guesswork needed.

Settings · Auto Start & Updates

Always ready — even after a reboot

In a restaurant environment, devices get power-cycled frequently. KDS is designed to come back online without any manual intervention — and to keep itself up to date with a single tap.

Auto Start on Boot
Enabling Auto Start on Boot launches KDS automatically when the Android device powers on. Tapping the toggle opens a system check dialog that reports the current permission status — auto-start permission and battery optimisation — and provides direct links to the relevant Android system settings to grant them.
Auto-start permission: Enabled · Battery permission: Disabled — links to system settings let staff fix it without navigating Android menus manually
Check for Updates
The System Information section shows the current app version and a Check for Updates button. Tapping it queries the update server and reports back immediately — either confirming the current version is up to date, or initiating an APK download and install flow directly within the app.
System Version: 1.4.2 · "Current version is up to date" — one tap to check, no app store required
Settings · Calling Screen

Connect a customer-facing Calling Screen

KDS can pair with a Calling Screen device over the local network. Once connected, it automatically pushes all active orders to the Calling Screen's "Preparing" section — and staff can move any order to "Ready" with a single tap.

1

Open device discovery

In Settings, tap the Calling Screen row to open the discovery panel. KDS scans the local network via mDNS and lists every Calling Screen device it finds — showing the device name, IP address, and port.

CS:edd2 found at 192.168.0.56:9090 — tap Connect to pair
2

Confirm the connection

After tapping Connect, a success dialog confirms the pairing with the device name and address. The connection is established over the local network with no manual IP entry required.

Connected to CS:edd2 · 192.168.0.56:9090 — confirmed and ready
3

Paired device shown in Settings

The connected Calling Screen appears as a Connected row in Settings. The Show Calling Button toggle enables a dedicated Call button on every order card. As soon as the device connects, KDS automatically pushes all orders currently on screen to the Calling Screen's Preparing section — no manual sync needed.

CS:edd2 connected · Show Calling Button toggled ON · existing orders auto-pushed to Preparing on connect
4

Call button moves order to Ready

With the Calling Button enabled, each order card shows both an All Done button and a Call button. Tapping Call sends a signal to the Calling Screen, moving that specific order from Preparing to Ready — notifying the customer that their order is available.

Each card has a Call button — one tap updates the order status on the Calling Screen in real time
📡

Calling Screen discovery uses mDNS (Bonjour) — no IP configuration or QR scanning. KDS finds compatible devices automatically as long as both devices are on the same Wi-Fi network.
